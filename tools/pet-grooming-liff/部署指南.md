## 第 1 步：在檔案 A 建「契約紀錄」工作表

1. 打開檔案 A（客戶資料那份 Sheets）
2. 底下分頁區 → 右鍵 → 新增工作表 → 命名為 **`契約紀錄`**（一字不差，程式靠這個名字找）
3. 在第 1 列填入這些標題（A-I 欄）：

| A | B | C | D | E | F | G | H | I |
|---|---|---|---|---|---|---|---|---|
| 時間戳 | 客戶電話 | 客戶姓名 | 寵物名 | 服務項目 | 金額 | PDF連結 | 哈希值 | LINE_userId |

4. 順便檢查「客戶資料」工作表的欄位，建議有這些 header（有的話程式會自動對應，沒有就跳過）：
   - `電話`（A 欄，必要）
   - `姓名`、`寵物名`、`品種`、`備註`、`建立時間`、`來源`、`本次服務`、`本次金額`、`LINE_userId`

> 💡 不需要全部都要建，但**電話**欄（A 欄）一定要在。其他欄位可後補，程式會容錯。

---

## 第 2 步：在 Google Drive 建「寵美契約存檔」資料夾

1. 打開 [Google Drive](https://drive.google.com)（用 `mayuolulu@gmail.com`）
2. 左側 → 新增 → 資料夾 → 命名 **`寵美契約存檔`**
3. 進入該資料夾 → 看網址列：
   ```
   https://drive.google.com/drive/folders/1ABC123xyz_LONG_STRING_HERE
   ```
   `/folders/` 後面那串就是**資料夾 ID**，**整段複製起來**。

## 第 3 步：建 LINE LIFF App

1. 登入 [LINE Developers Console](https://developers.line.biz/console/)
2. 選 Provider（沒有就建一個叫「寵美工具」）
3. 進入你的**官方帳號 Channel**（Messaging API 類型的那個）
4. 上方頁籤切到「**LIFF**」
5. 按「**Add**」新增：
   - **LIFF app name**：`寵美資訊登記`
   - **Size**：`Full`
   - **Endpoint URL**：
     ```
     https://live90200-elio.github.io/mylittlethings/tools/pet-grooming-liff/
     ```
   - **Scope**：勾 `profile` + `openid`（需要讀客戶名字）
   - **Bot link feature**：`On (Aggressive)` 或 `Off` 都可（官方帳號相關，不影響表單）
   - **Scan QR**：保持預設
6. 建好後會拿到一個 **LIFF ID**（格式類似 `2001234567-abCdEfGh`）→ **整串複製起來**

---

## 第 4 步：貼 Apps Script 程式 + 設定

1. 打開 [Apps Script](https://script.google.com/) → 新專案
2. 專案命名：`寵美客戶 LIFF 表單`
3. 左邊 `Code.gs` → 全選刪掉 → 貼上 `tools/pet-grooming-liff/Code.gs` 的**完整內容**
4. 改第 20-27 行的設定：
   - `FILE_A_URL`：已預填寵美檔案 A 網址（若 ID 不同要改）
   - `PDF_FOLDER_ID`：貼上第 2 步拿到的資料夾 ID
   - `LIFF_KEY`：改成你自己的亂數密碼（16+ 字元，越亂越好）
   - `SHOP_NAME` / `SHOP_ADDRESS` / `SHOP_TEL`：店家資訊（PDF 抬頭用）
5. Ctrl+S 儲存
6. 上方選單 **執行** → 選 `testDoPost` 函式 → 點「執行」
   - 第一次會彈權限：Sheets + Drive → **全部授權**
   - 執行完看「執行記錄」→ 應該出現 `{"ok":true,"pdfUrl":"...","hash":"..."}`
   - 去 Drive「寵美契約存檔」資料夾 → 應該看到一個測試 PDF（檔名含 `0912345678_測試寵物_...`）
   - 去 Sheets「契約紀錄」工作表 → 應該多一列測試資料
   - 去 Sheets「客戶資料」工作表 → 應該多一筆 `0912345678`
7. **測完把測試資料刪掉**（測試 PDF / 契約紀錄那列 / 客戶資料那列）

## 第 5 步：部署 Apps Script 為 Web App

1. 右上角「**部署**」→「**新增部署作業**」
2. 齒輪圖示 → 選「**網頁應用程式**」
3. 設定：
   - **說明**：`寵美 LIFF v1`
   - **執行身分**：**我（mayuolulu@gmail.com）**
   - **誰可以存取**：**所有人**（⚠️ 一定要選，不然 LIFF 會被擋）
4. 按「**部署**」→ 同意權限
5. 會出現「**網頁應用程式網址**」（`.../exec` 結尾）→ **整串複製起來**

## 第 6 步：把 LIFF ID 和 API URL 填進 index.html

1. 編輯 `tools/pet-grooming-liff/index.html`
2. 找到 210 行左右的「⚠️ 必改區」：
   ```javascript
   const LIFF_ID = "LIFF_ID_PLACEHOLDER";
   const LIFF_API_URL = "LIFF_API_URL_PLACEHOLDER";
   const LIFF_KEY = "change-me-to-random-long-string";
   ```
3. 分別替換：
   - `LIFF_ID` → 第 3 步拿到的 LIFF ID
   - `LIFF_API_URL` → 第 5 步拿到的 `.../exec` 網址
   - `LIFF_KEY` → 跟 Apps Script 裡**一模一樣**的密碼
4. 儲存

## 🐛 常見問題

**Q1：LIFF 送出時跳「送出失敗：Failed to fetch」**
→ Apps Script 沒部署為「所有人可存取」，或 LIFF_API_URL 填錯。重開部署流程，新增部署，選「所有人」。

**Q2：LIFF 一開就白畫面 / 錯誤**
→ LIFF Endpoint URL 跟實際網址不一致（通常是少了尾斜線）。改成 `.../pet-grooming-liff/`（結尾要有 `/`）。

**Q3：送出成功但沒看到 PDF 連結**
→ `PDF_FOLDER_ID` 填錯或資料夾沒權限。去 Apps Script 執行 `testDoPost`，看執行記錄的錯誤。

**Q4：PDF 裡中文變框框**
→ Apps Script 產 PDF 時字體問題，已用 Noto Sans CJK TC fallback。如果還有問題，改成 `PingFang TC` 或用內建 `sans-serif`。

**Q5：換了密碼，LIFF 送出就報 auth_failed**
→ index.html 的 `LIFF_KEY` 和 Code.gs 的 `LIFF_KEY` 必須**完全一致**。檢查有沒有空白或大小寫錯。

---

## 🔐 安全提醒

- `LIFF_KEY` 是公開倉庫看得到的佔位符 `change-me-to-random-long-string`
- **真實密碼只存在你的 Apps Script 線上編輯器 + 你本地改好但別 commit 的 index.html**
- 如果不小心 commit 了真實密碼 → 立刻改密碼、重新部署 Apps Script、改 index.html、再 commit 一次新佔位符
- 參考預約小幫手的做法：本地改密碼後**不要 commit 那行**，git 用 `git update-index --skip-worktree` 可以忽略該檔的本地改動
