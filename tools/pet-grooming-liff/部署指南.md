# 寵美客戶 LIFF 表單 — 部署指南

> 這份是**照著做**的流程清單。全部照順序走完大約 30-45 分鐘。
> 有任何一步卡住，直接回 Claude 問。

> 🏗️ **架構**：
> - **前端 `index.html`**：靜態檔，放 GitHub Pages（或任何靜態主機）。LIFF_ID + Apps Script API URL 寫死在檔案裡（都不是機密）。
> - **後端 `Code.gs`**：Apps Script 當純 API，只處理 POST。
> - **沒有共用 KEY**，也沒有模板注入。

## 第 1 步：在檔案 A 建「契約紀錄」工作表

1. 打開檔案 A（客戶資料那份 Sheets）
2. 底下分頁區 → 右鍵 → 新增工作表 → 命名為 **`契約紀錄`**（一字不差，程式靠這個名字找）
3. 在第 1 列填入這些標題（A-I 欄）：

| A | B | C | D | E | F | G | H | I |
|---|---|---|---|---|---|---|---|---|
| 時間戳 | 客戶電話 | 客戶姓名 | 寵物名 | 服務項目 | 金額 | PDF連結 | 哈希值 | LINE_userId |

4. 順便檢查「客戶資料」工作表的欄位，建議有這些 header（有的話程式會自動對應，沒有就跳過）：
   - `電話`（A 欄，必要）
   - `姓名`、`寵物名`、`品種`、`備註`、`建立時間`、`來源`、`本次服務`、`本次金額`、`LINE_userId`

> 💡 不需要全部都要建，但**電話**欄（A 欄）一定要在。其他欄位可後補，程式會容錯。

---

## 第 2 步：在 Google Drive 建「寵美契約存檔」資料夾

1. 打開 [Google Drive](https://drive.google.com)（用 `mayuolulu@gmail.com`）
2. 左側 → 新增 → 資料夾 → 命名 **`寵美契約存檔`**
3. 進入該資料夾 → 看網址列：
   ```
   https://drive.google.com/drive/folders/1ABC123xyz_LONG_STRING_HERE
   ```
   `/folders/` 後面那串就是**資料夾 ID**，**整段複製起來**。

---

## 第 3 步：建 Apps Script 專案 + 貼 `Code.gs`

1. 打開 [Apps Script](https://script.google.com/) → 新專案
2. 專案命名：`寵美客戶 LIFF 表單 API`
3. 左邊 `Code.gs` → 全選刪掉 → 貼上 `tools/pet-grooming-liff/Code.gs` 完整內容
4. 改 `Code.gs` 上方設定區：
   - `FILE_A_URL`：已預填寵美檔案 A 網址（若 ID 不同要改）
   - `PDF_FOLDER_ID`：貼上第 2 步拿到的資料夾 ID
   - `SHOP_NAME` / `SHOP_ADDRESS` / `SHOP_TEL`：店家資訊（PDF 抬頭用）
5. Ctrl+S 儲存

> 💡 這版的 Code.gs **不用**管 `LIFF_ID` / `LIFF_KEY`。前端 HTML 自己處理 LIFF，後端只當 API。

---

## 第 4 步：部署 Apps Script 為 Web App

1. 右上角「**部署**」→「**新增部署作業**」
2. 齒輪圖示 → 選「**網頁應用程式**」
3. 設定：
   - **說明**：`寵美 LIFF API v1`
   - **執行身分**：**我（mayuolulu@gmail.com）**
   - **誰可以存取**：**所有人**（⚠️ 一定要選，不然前端 fetch 會被擋）
4. 按「**部署**」→ 同意權限（Sheets / Drive 第一次會彈）
5. 會出現「**網頁應用程式網址**」（`.../exec` 結尾）→ **整串複製起來**

> 🔄 **之後改程式要重新部署時**，選「**管理部署作業**」→ 鉛筆圖示「**編輯**」→ 版本選「**新版本**」→ 部署。這樣網址會保持不變。

## 第 5 步：把 API URL 填進 `index.html`

1. 打開 `tools/pet-grooming-liff/index.html`
2. 找到檔案下半部的設定區：
   ```js

   const LIFF_API_URL = "https://script.google.com/macros/s/.../exec";
   ```
3. `LIFF_API_URL` 換成第 4 步拿到的 `/exec` 網址

1. 打開 [live90200-Elio/mylittlethings](https://github.com/live90200-Elio/mylittlethings) → Settings → Pages

3. Branch 選 `master` → 資料夾選 `/ (root)` → Save

5. **你的 LIFF 前端網址**就是：

   ```
   https://live90200-elio.github.io/mylittlethings/tools/pet-grooming-liff/
   ```
   **整串複製起來**，下一步要填。

## 第 7 步：建 LINE LIFF App

1. 登入 [LINE Developers Console](https://developers.line.biz/console/)
2. 選 Provider（沒有就建一個叫「寵美工具」）
3. 進入你的**官方帳號 Channel**（Messaging API 類型的那個）
4. 上方頁籤切到「**LIFF**」
5. 按「**Add**」新增：
   - **LIFF app name**：`寵美資訊登記`
   - **Size**：`Full`
   - **Endpoint URL**：貼第 6 步拿到的 GitHub Pages 網址（**不用加尾斜線**）
   - **Scope**：勾 `profile` + `openid`（需要讀客戶名字）
   - **Bot link feature**：`On (Aggressive)` 或 `Off` 都可
   - **Scan QR**：保持預設
6. 建好後會拿到一個 **LIFF ID**（格式類似 `2001234567-abCdEfGh`）→ **整串複製起來**

## 第 8 步：把 LIFF ID 填進 `index.html`

2. 找到第 5 步同一塊設定區的 `LIFF_ID`，改成第 7 步拿到的 ID
3. 儲存、commit、push
4. GitHub Pages 會自動重建（約 1 分鐘）

## 第 9 步：先用 `testDoPost` 驗證後端

1. Apps Script 編輯器上方 **執行** → 選 `testDoPost` 函式 → 點「執行」
2. 第一次會彈權限：Sheets + Drive → **全部授權**
3. 執行完看「執行記錄」→ 應該出現 `{"ok":true,"pdfUrl":"...","hash":"..."}`
4. 驗證：
   - [ ] Drive「寵美契約存檔」有測試 PDF（檔名含 `0912345678_測試寵物_...`）
   - [ ] Sheets「契約紀錄」多一列測試資料
   - [ ] Sheets「客戶資料」多一筆 `0912345678`
5. **測完把測試資料全刪掉**（PDF + 兩個工作表的測試列）

## 第 10 步：在 LINE 內測試一筆真實送出

1. 打開 LINE → 加你自己的官方帳號好友
2. 訊息欄貼連結：`https://liff.line.me/<你的_LIFF_ID>` → 送出
3. 點自己送出的訊息中的連結 → 會在 LINE 內開啟 LIFF
4. 填測試資料（姓名、電話、寵物名、勾選項目、打勾同意、簽名）→ 送出
5. 應該會看到「✅ 已送出」畫面 + PDF 連結
6. 驗證：
   - [ ] Sheets「契約紀錄」有新一列
   - [ ] Sheets「客戶資料」有新客戶
   - [ ] Drive「寵美契約存檔」有新 PDF，打開看格式對不對

## 🐛 常見問題

**Q1：LIFF 送出時跳「送出失敗：Failed to fetch」**
→ Apps Script 沒部署為「所有人可存取」，或 `LIFF_API_URL` 填錯。檢查 `index.html` 的 `LIFF_API_URL` 是不是 `/exec` 結尾的完整網址。

**Q2：LIFF 一開就白畫面 / 錯誤**
→ LIFF Endpoint URL 跟 GitHub Pages 實際網址不一致。回第 7 步編輯 LIFF，確認 Endpoint URL 是 `https://live90200-elio.github.io/mylittlethings/tools/pet-grooming-liff/`。

**Q3：改了 `index.html` 推到 GitHub 但沒生效**
→ GitHub Pages 每次 push 會自動重建，但要等 1-2 分鐘。強制刷新（Ctrl+Shift+R）或在 Console 看 Network tab 有沒有拿到新版本。

**Q4：送出成功但沒看到 PDF 連結**
→ `PDF_FOLDER_ID` 填錯或資料夾沒權限。去 Apps Script 執行 `testDoPost`，看執行記錄的錯誤。

**Q5：PDF 裡中文變框框**
→ Apps Script 產 PDF 時字體問題，已用 Noto Sans CJK TC fallback。如果還有問題，改成 `PingFang TC` 或用內建 `sans-serif`。

**Q6：API 送出噴 `missing_liffUserId`**
→ 前端沒登入 LINE 就送（例如直接用瀏覽器開 GitHub Pages 網址）。這是預期行為：必須在 LINE 內開 LIFF 才能送出。

---

## 🔐 安全提醒

- **`LIFF_ID` 跟 Apps Script `/exec` URL 都不是機密**：LIFF_ID 在 LINE Console 就能查；Apps Script `/exec` URL 是 Web App 公開網址。寫在 public repo 完全沒問題。
- **真正的防線**：
  - 後端必填 `liffUserId`，直接打 API 沒帶會被擋
  - Apps Script 只接受特定欄位，不會因為來路不明的 POST 壞資料
- **想更嚴**：V2 可改成前端帶 `liff.getAccessToken()`，後端呼叫 LINE API 驗 token 真實性。
