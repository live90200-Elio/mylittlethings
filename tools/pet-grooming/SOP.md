# 寵美管理工具 — LINE + Google 雲端管理系統 SOP

## 第一階段：系統建置（打好地基）

### 1. Google 試算表架構：雙層過濾機制

為了確保員工看不到總營業額，需要建立兩份獨立的試算表。

**檔案 A：【員工服務紀錄表】（開放員工編輯權限）**
- 工作表一：今日服務
  - 欄位：寵物名、主人電話、服務項目（洗/剪）、金額、付款方式（現金/儲值扣抵）、負責美容師
- 工作表二：儲值金查詢
  - 欄位：客戶姓名、剩餘餘額
  - 員工結帳時用 Ctrl+F 快速搜尋

**檔案 B：【老闆私藏報表】（僅限老闆開啟）**
- 使用 `IMPORTRANGE` 自動抓取「檔案 A」所有數據
- 計算：月總營收、各美容師業績拆帳、耗材支出、利潤分析

### 2. LINE 官方帳號 (OA) 優化設定

- **圖文選單**：六宮格或三宮格按鈕（預約、價目表、作品集）
- **多頁訊息**：儲值方案介紹，客戶隨時點選查看

---

## 第二階段：資料轉移（告別紙本）

利用離峰時間處理：
- **結清紙本**：紙本客戶資料、最後一次美容備註（如：耳朵易發炎）、儲值金餘額 → 輸入到「儲值金查詢」分頁
- **建立 ID**：以「手機號碼」作為唯一索引，避免同名同姓混亂

---

## 第三階段：每日運作 SOP（員工執行）

### 步驟 1：客戶預約
客戶 LINE 詢問 → 員工回覆確認 → 立即填入【今日服務】（或 Google 日曆）
- 優點：所有員工打開手機就能看到今天還有幾隻狗要洗

### 步驟 2：接到寵物
搜尋試算表客戶備註 → 確認毛孩狀況 → LINE 傳「寶貝已到店」預設訊息

### 步驟 3：結帳與記錄
- 儲值扣抵：試算表扣除餘額 + 填寫當日消費
- 現金/轉帳：直接填付款方式欄位
- **重要動作**：LINE 傳「今日消費 X 元，儲值金剩餘 Y 元」給客戶當存證，取代實體會員卡

---

## 第四階段：老闆管理流程（隱私防護）

### 1. 權限物理隔離
- 店內公用平板/電腦：只登入「員工服務紀錄表」帳號
- 老闆個人手機/電腦：登入自己帳號看「老闆私藏報表」
- 關鍵：員工看到的只是「一筆一筆流水帳」，忙工作時很難去統計月加總

### 2. 數據核對
每週花 15 分鐘打開【老闆私藏報表】，核對與現場現金是否相符

## 執行細節建議表

| 項目 | 具體做法 |
|---|---|
| 防弊機制 | 每筆儲值扣抵後，員工必須在 LINE 傳訊息給客戶當存證 |
| 美容備註 | 試算表增加「重要提醒」欄位（如：兇、心臟病），變色提醒 |
| 月營收隱藏 | 員工表的總金額欄位不要設定 SUM 公式，只留空白 |
| LINE 標籤 | 後台為客戶貼「有儲值」、「剪毛常客」標籤，精準推播 |

## 工作拆解：哪些 Claude 幫得上、哪些要自己做

### 🙋 你自己做（Claude 幫不了）
- 建兩份 Google 試算表 + `IMPORTRANGE` 公式
- LINE OA 後台圖文選單、多頁訊息、客戶標籤
- 紙本資料輸入 Google 試算表
- 店內平板 / 老闆手機的 Google 帳號登入隔離

### 🤖 Claude 可以做成網頁工具（跑在 GitHub Pages）
- **儲值金查詢頁**（客戶用）：客戶輸入手機號 → 看自己的餘額 + 近期消費
- **員工快速查詢頁**（店內平板用）：輸入手機號 → 一頁看完客戶備註 + 寵物資料 + 餘額
- **結帳小幫手**（員工用）：輸入消費金額 → 自動產生要傳給客戶的 LINE 訊息文字（一鍵複製）
- **SOP 備忘單**（員工用）：把上面 4 個階段做成漂亮的網頁，新人入職看

> 💡 這些網頁工具要讀寫 Google 試算表的資料，會需要搭 Google Apps Script 做 API。EP 系列之後的影片應該有教。
