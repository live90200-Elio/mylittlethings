# Mylittlethings 使用教學

> 寵物美容店所有自動化工具的「誰該做什麼」一站式說明。
> 任何角色照著這份做，就不會打架到別人的工作。

**建議放置**：
- 員工平板：把這份書籤起來
- 老闆手機：書籤 + 列印「老闆」+「員工」兩段貼櫃台
- 維護工程師（Elio）：當 runbook，出狀況時翻

---

## 🐾 客戶（用 LINE 填表單）

### 第一次來店時要做的事
1. **加官方帳號好友**：店員會出示 QR code
2. **在聊天視窗點「填新客戶表單」**（豐富選單上的按鈕）
3. 表單會自動帶入你的 LINE 顯示名稱
4. 填寫：
   - 客戶姓名（如果跟 LINE 名稱不同可以改）
   - 手機號碼
   - 寵物名、品種
   - 美容備註（過敏、剪毛習慣等）
   - 勾選本次服務項目（洗澡 / 修頭 / 全套 / 剪指甲...）
   - 金額（不確定就留空，現場確認）
5. 點「同意以下注意事項」
6. 在指定區塊**用手指簽名**
7. 送出
8. 等 5–10 秒，看到「✅ 簽約完成」+ 「查看 PDF 契約」連結

### 注意事項
- ⚠️ 一定要**在 LINE 內**開表單（不是用 Safari / Chrome 直接打網址）
- 簽完名要看清楚再送，**送出後就不能改**
- 送出後老闆會即時收到通知，準備服務
- 沒收到 PDF 連結 = 送出失敗，請重新打開填一次（資料不會自動保留）
- 之後每次來店**不用再填**，店員會直接幫你記帳

## 👔 老闆（每天的事）

### 自動會發生的事（不用做什麼，看通知就好）

| 時間 | 收到的通知 | 你要做的事 |
|---|---|---|
| 16:00 | LINE「📅 明日預約清單」 | 確認；有疑慮主動聯絡客戶 |
| 客戶簽 LIFF 即時 | LINE「✅ 新簽約完成」+ PDF 連結 | 點 PDF 連結確認；列印或現場補美容師簽名 |
| 20:00 | LINE「📊 營收報表」+ Email 完整版 | 看 Email 明細表格、確認金額正確 |
| 服務掛掉 | LINE「⚠️ Apps Script 服務異常」 | 進 Apps Script 管理部署看狀況；同事故 2 小時內只警報一次 |

### 主動要做的事

#### 每天晚上：建明日預約事件
打開 **Google 日曆**，每筆預約建一個事件：
- **標題**：寵物名 + 服務記號
  - 剪毛：加「✂️」或「剪」字（例：`小白✂️` 或 `小白剪`）
  - 純洗澡：不加（例：`小白`）
- **描述欄**：客戶手機號（格式不限：`0905-435-751` / `0905435751` / `+886-905-435-751` 都會自動辨識）
- **店休那天**：標題寫「特休」「公休」「休假」「店休」其中一個 → 推播會自動跳過

#### 不定期：補客戶資料
去**檔案 A「客戶資料」分頁**手動補這幾欄（LIFF 不會寫這些）：

| 欄 | 標題 | 範例 |
|---|---|---|
| G | 重要提醒 | `兇；心臟病；怕剪刀聲`（分號分隔） |
| I | 儲值金餘額 | `3200` |
| J | 最近到店 | `2026-04-25` |

### 老闆要記住的紅線
- ⚠️ LINE 必須**加自己的官方帳號為好友**（不加，所有 Push 推不到）
- ⚠️ **檔案 A 永遠建在老闆帳號**，不要建在員工帳號（員工離職你會失去資料所有權）
- ⚠️ **「服務紀錄」分頁 E 欄（金額）絕對不能放 `=SUM()`** — 員工會看到月營收，違反設計
- ⚠️ 想看月收入？看**老闆儀表板（PWA）**或**檔案 B**，不是看檔案 A
- ⚠️ 客戶資料分頁的 A 欄電話、B 欄姓名是主鍵，**不要手動刪除整列**（會破壞 LIFF upsert）
- ⚠️ K-O 欄（建立時間 / 來源 / 本次服務 / 本次金額 / LINE_userId）是 LIFF 自動寫的，不要動

## 👩‍🔧 員工（店內平板用）

### 平板登入
- **只**登「店共用帳號」（例：`shop-tablet@gmail.com`）
- ❌ **不要**登老闆帳號
- 書籤兩個頁面：「員工查詢頁」+「檔案 A 試算表」

### 客人來店流程
1. 開**員工查詢頁** → 輸入主人手機號
2. 看到：客戶姓名 / 寵物 / 重要提醒 / 美容備註 / 儲值金餘額 / 最近到店
3. 服務完成 → 切到**檔案 A → 「服務紀錄」分頁** → 加一列：

| A 日期 | B 主人電話 | C 寵物名 | D 服務項目 | E 金額 | F 付款方式 | G 美容師 | H 備註 |
|---|---|---|---|---|---|---|---|
| 2026-04-26 | 0912345678 | 小白 | 全套 | 800 | 現金 | Amy | 加耳道清潔 |

### 員工要記住的紅線
- ⚠️ **服務紀錄當天填完**：晚上 20:00 自動跑營收報表會用這欄資料，沒填數字會錯
- ⚠️ **金額 E 欄填純數字**（`800`，不是 `NT$800`、不是 `800元`）
- ⚠️ **絕對不要在「服務紀錄」分頁底下加 SUM 公式**（這是老闆給的禁忌）
- ⚠️ **客戶資料分頁只能改 G 欄（重要提醒）、H 欄（美容備註）**，其他欄位不要動
- ✅ 客人說他在 LINE 填過表單 → 客戶資料分頁應該已經自動有他了，不用手動建
- 🤔 客人說他來剪過但找不到資料 → 可能 LIFF 留錯電話 → 跟老闆反應，老闆來修

## 🛠️ 維護工程師（Elio 自己 — runbook）

### 主要 KEY / Token / ID 一覽表

| 名稱 | 類型 | 存在哪 | 進 git？ | 用途 |
|---|---|---|---|---|
| `LINE_TOKEN` | 🔒 密碼 | ScriptProperties（3 個專案各一份） | ❌ | LINE Push 認證 |
| `BOSS_USER_ID` | 🔒 隱私 | ScriptProperties（3 個專案各一份） | ❌ | 推播對象 |
| `APPT_KEY` | 🔒 密碼 | ScriptProperties（appointment-helper 獨有） | ❌ | 預約頁登入密碼 |
| `BOSS_EMAIL` | 🔒 隱私 | ScriptProperties（pet-grooming-ops 獨有） | ❌ | 收每日營收報表 |
| `BOSS_KEY` | 🔒 密碼 | 老闆儀表板 Apps Script 線上常數 | ❌（本地佔位符） | 儀表板登入密碼 |
| `HEALTH_URL` | 公開 | ScriptProperties（pet-grooming-ops） | ❌（純為了好改） | 健康檢查目標清單 |
| `LIFF_ID` | 🔓 公開 | `tools/pet-grooming-liff/index.html` | ✅ | LINE 平台識別 |
| `PDF_FOLDER_ID` | 📎 ID | `tools/pet-grooming-liff/Code.gs` | ✅ | Drive 契約資料夾 |
| `FILE_A_URL` | 📎 ID | 多支 Code.gs（appointment-helper、pet-grooming-liff、pet-grooming-ops） | ✅ | 主資料 Sheets |

> 區分原則：**密碼 / 隱私 → ScriptProperties**；**ID / URL → 程式碼常數**（踩坑筆記第 169 條）

### 哪三個專案要設 LINE_TOKEN + BOSS_USER_ID
1. `appointment-helper` — 流程 1 預約推播
2. `pet-grooming-liff` — 流程 2 簽約即時通知
3. `pet-grooming-ops` — 流程 3 營收報表 + 流程 4 健康監控

換新 token 要 3 個地方都更新。

### Apps Script 專案地圖

| 專案 | 部署形式 | 主要函式 | Trigger |
|---|---|---|---|
| `pet-grooming` | Web App `/exec` | doGet 客戶資料 | 無 |
| `pet-grooming-boss` | Web App `/exec` | doGet 儀表板資料 | 無 |
| `appointment-helper` | Web App `/exec` | doGet 預約 + health=1、`dailyPushTomorrowAppt` | 16:00 daily |
| `pet-grooming-liff` | Web App `/exec` | doPost 簽約 + doGet health=1、`notifyBossNewContract` | 無（即時觸發） |
| `pet-grooming-ops` | **純 trigger，不部署** | `dailyRevenueReport`、`healthCheck` | 20:00 daily + 每小時 |

### 檔案 A Sheets 分頁速查

| 分頁 | 欄位 | 寫入者 |
|---|---|---|
| 客戶資料 | A-O 15 欄 | LIFF doPost upsert + 老闆手填 |
| 服務紀錄 | A-H 8 欄 | 員工手填 |
| 儲值紀錄 | A-F 6 欄 | 老闆手填（選配） |
| 契約紀錄 | A-I 9 欄 | LIFF doPost append |
| 簽約通知 log | A-H 8 欄 | LIFF notifyBoss append |

### 出問題時的排查順序

| 症狀 | 先看哪 | 接著怎麼辦 |
|---|---|---|
| 老闆完全沒收任何 LINE 通知 | 老闆是不是還是官方帳號好友？ | LINE_TOKEN 是否過期 → 三個專案 ScriptProperty 重設 |
| 客戶簽 LIFF，老闆沒收通知 | 「簽約通知 log」分頁有沒有 append | 沒 append → doPost 改完沒重部署；有但 status=error → 看 H 欄 errorMsg |
| 16:00 預約沒推 | Apps Script 觸發條件記錄查當天執行 | 失敗 → 看執行記錄；沒記錄 → 時區是不是 Asia/Taipei |
| 20:00 營收沒推或數字錯 | `testRevenueStats` 看計算過程 | 「服務紀錄」A 欄是不是 Date 物件（純文字會被跳過 → 數字偏低） |
| 健康監控狂發警報 | 警報訊息的 service label 看是哪個專案 | 進該專案管理部署看執行記錄 |
| LIFF 改完手機還是舊版 | 不是程式問題，是 LINE app 快取 | 手機設定 → 應用程式 → LINE → 儲存空間 → **清除快取**（不是清除資料） |
| 員工查詢頁查不到客戶 | 主人電話有沒有 normalize 到 09 開頭 10 碼 | 客戶資料 A 欄電話格式是否正確 |
| 改了 Code.gs 但行為沒變 | 有沒有「管理部署 → 新版本 → 部署」 | 純 trigger 專案不用部署，改完即生效；Web App 必須重部署 |
| Mail 寄不出去 | 第一次沒授權 | 編輯器手動執行該函式一次，會跳授權對話框 |

### 兩個關鍵規則別忘

1. **改 ScriptProperty 不需要重部署** — 所有 trigger 跟 doGet/doPost 下次跑時即時讀新值
2. **改 Code.gs 必須重部署** — Web App URL 永遠跑「上次部署的快照」，不是編輯器最新版。trigger 倒是即時用新 code。所以動到 doGet/doPost 一定要：右上「部署」→「管理部署作業」→「鉛筆」→「版本：新版本」→「部署」。部署 URL 不會變，前端不用改。

### 常見炸雷

1. **LINE_TOKEN 過期** → LINE Developers Console 重發 → 三個專案 ScriptProperty 都要同步
2. **APPT_KEY 改了** → appointment-helper ScriptProperty + 員工/老闆手機記住的密碼都要更新
3. **檔案 A spreadsheet 換 URL** → 三支 Code.gs 的 `FILE_A_URL` 常數要全改 + 線上重貼 + 重部署
4. **PDF_FOLDER_ID 改了** → pet-grooming-liff Code.gs 改 + 重部署（不是密碼，可以直接寫進 git）
5. **「服務紀錄」分頁改了欄位順序** → `pet-grooming-ops/Code.gs` 的 `computeRevenueStats` 內 `r[0]~r[7]` 對應要跟著改
6. **LIFF_ID 字元 1/l/I 混淆** → 永遠 copy-paste，不要靠眼睛校對（踩坑筆記第 168 條）
7. **貼新版 Code.gs 蓋掉線上密碼** → 貼之前先看本地 Code.gs 對應行有沒有佔位符，有就先去 ScriptProperty 設好真值再貼

### 完整收工 checklist

- [ ] 線上 Apps Script 跟本地 Code.gs 同步（除了 ScriptProperty 那些值）
- [ ] 動到 doGet / doPost 的專案有重部署新版本
- [ ] 本地有改的檔案都 commit + push GitHub
- [ ] 更新 Obsidian 工作筆記（`secondbrain/Mylittlethings/工作筆記.md`）
