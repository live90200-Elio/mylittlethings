# 老闆儀表板 部署指南

> 目的：把 `index.html` 從「無資料」狀態接上真實交易紀錄。
> 前置：已部署完員工查詢頁（檔案 A 的「客戶資料」工作表已在用）。
> 預計時間：15 分鐘（含在檔案 A 新建「服務紀錄」工作表）。

---

## 🎯 總流程

```
檔案 A 新增「服務紀錄」工作表
  → 建檔案 B（空的試算表）
    → 檔案 B 內打開 Apps Script
      → 貼上 Code.gs
        → 改兩個常數（FILE_A_URL、BOSS_KEY）
          → 部署為網頁應用程式（任何人）
            → 拿到 /exec 網址
              → 貼到 index.html 的 BOSS_API_URL
                → 完成 ✅
```

這裡的「檔案 A／檔案 B」為什麼要分兩份？
- **檔案 A**（員工用）：客戶資料 + 服務紀錄，員工有編輯權。
- **檔案 B**（老闆用）：只掛 Apps Script，員工看不到。BOSS_KEY 密碼也只寫在這份腳本裡。

---

## 步驟 1：在檔案 A 新增「服務紀錄」工作表

1. 用**老闆帳號**打開【檔案 A：員工服務紀錄表】
2. 底部「+」新增工作表 → 名稱改成 **`服務紀錄`**（剛好 4 字、不要有空白）
3. 第 1 列填欄位標題：

   | A | B | C | D | E | F | G | H |
   |---|---|---|---|---|---|---|---|
   | 日期 | 主人電話 | 寵物名 | 服務項目 | 金額 | 付款方式 | 負責美容師 | 備註 |

4. 選整個 A 欄 → **格式 → 數字 → 日期**（確保日期欄讀得出來）
5. 選整個 E 欄 → **格式 → 數字 → 數字**（金額不要被當成文字）
6. 先填 2-3 筆假資料測試用，例如：

   | 日期 | 主人電話 | 寵物名 | 服務項目 | 金額 | 付款方式 | 負責美容師 | 備註 |
   |------|----------|--------|----------|------|----------|------------|------|
   | 2026-04-10 | 0912345678 | Momo | 全洗 | 800 | 現金 | 阿珍 |  |
   | 2026-04-15 | 0911222333 | 豆花 | 洗+剪 | 1200 | 儲值 | 小婷 |  |
   | 2026-04-20 | 0912345678 | Momo | 剪毛 | 600 | LINE Pay | 阿珍 |  |

> ⚠️ **金額欄絕對不要放 `=SUM()`**。員工也能看到這份工作表，月營收必須隱藏（SOP 規定）。

## 步驟 2：建檔案 B（老闆私藏）

1. **用老闆帳號**到 [sheets.new](https://sheets.new) 建一份空白試算表
2. 命名為【**檔案 B：老闆私藏報表**】
3. 裡面不用填任何東西（這份純粹是 Apps Script 的容器）
4. **不要分享給員工**

> 為什麼要另開一份？因為 BOSS_KEY 密碼會寫在 Apps Script 裡。如果把腳本掛在檔案 A，員工就能從「擴充功能→Apps Script」看到密碼。檔案 B 只有老闆有權限，密碼才安全。

## 步驟 3：檔案 B 內打開 Apps Script，貼上 Code.gs

1. 在檔案 B 裡：**擴充功能 → Apps Script**
2. 打開 `tools/pet-grooming-boss/Code.gs`（這個 repo 內），全選複製
3. 刪掉預設的 `function myFunction() {}`，貼上整份 Code.gs
4. 左上專案名改成例如 `老闆儀表板 API`
5. `Ctrl+S` 先存檔（還不能執行，下一步要先改兩個常數）

## 步驟 4：改兩個常數（⚠️ 必改）

Code.gs 第 22-25 行：

```js
const FILE_A_URL = "https://docs.google.com/spreadsheets/d/YOUR_FILE_A_ID/edit";
const BOSS_KEY = "change-me-to-random-long-string";
```

### 4-1 改 `FILE_A_URL`

1. 另開一個分頁，打開【檔案 A】
2. 從網址列整段複製（包含 `https://` 到 `/edit` 的結尾）
3. 貼回 Code.gs，取代 `YOUR_FILE_A_ID` 那一整串

範例：
```js
const FILE_A_URL = "https://docs.google.com/spreadsheets/d/1aB2cDeFgHiJkLmNoPqRsTuVwXyZ0123456789/edit";
```

### 4-2 設 `BOSS_KEY`（老闆的密碼）

- 長度至少 **12 個字元**，建議 16+
- 建議組合：亂碼 + 只有你知道的詞（不要用生日、手機後四碼）
- 不能有空白、換行
- **記到你的密碼管理器裡，不要只存在腦中**

範例（請用自己的）：
```js
const BOSS_KEY = "PetShop-2026-Q2-xK9mL3";
```

> 🔒 為什麼這個叫「密碼」就夠了？
> Apps Script 部署時存取權設「任何人」，但 Code.gs 裡會檢查 `params.key === BOSS_KEY`。密碼不對就回 `auth_failed`。所以網址外流也沒關係，只要 BOSS_KEY 沒外流，別人拿到 URL 也看不到數字。

改完 `Ctrl+S` 存檔。

## 步驟 5：測試一次確定沒錯

1. 函式下拉選單 → 選 `testDoGet`
2. 按「執行」
3. **第一次會彈出授權提示**：
   - 按「檢閱權限」→ 選老闆帳號
   - 看到「Google 尚未驗證此應用程式」→ 按「**進階**」→「前往 [你的專案名]（不安全）」
   - 按「允許」
   > 🤔「不安全」？因為這腳本是你自己寫的沒送 Google 審查。你的帳號、你的腳本，OK 的。
4. 執行記錄應該看到類似：
   ```json
   {"period":{"from":"2026-04-01","to":"2026-04-30","label":"2026 年 4 月"},"totalRevenue":2600,"transactionCount":3,"avgPerTransaction":867,...}
   ```
5. 如果看到 `"error":"找不到工作表：服務紀錄"` → 回步驟 1 確認工作表叫什麼名字
6. 如果看到 `"error":"auth_failed"` → 正常，testDoGet 裡有帶 key，不會看到；如果在網頁看到才處理

## 步驟 6：部署為網頁應用程式

1. 右上「**部署**」→「**新增部署**」
2. 齒輪 → 選「**網頁應用程式**」
3. 設定：
   - 說明：`v1`
   - 執行身分：**我（老闆帳號）**
   - 存取權：**任何人**（⚠️ 不是「任何擁有 Google 帳戶的使用者」）
4. 按「部署」
5. 複製「**網頁應用程式 URL**」：
   ```
   https://script.google.com/macros/s/AKfycbx.....BOSS...../exec
   ```
6. 按「完成」

## 步驟 7：把 URL 貼進 index.html

打開 `tools/pet-grooming-boss/index.html`，找到第 **191 行**：

```js
const BOSS_API_URL = "";
```

改成：

```js
const BOSS_API_URL = "https://script.google.com/macros/s/AKfycbx.....BOSS...../exec";
```

⚠️ **不要把 BOSS_KEY 寫在這裡**。BOSS_KEY 只有打開網頁時老闆手動輸入，不進原始碼、不進 GitHub。

存檔。

## 步驟 8：驗證

1. 打開 index.html（雙擊或丟上 GitHub Pages）
2. 看到登入畫面 → 輸入你在步驟 4-2 設的 BOSS_KEY
3. 看到 KPI 卡、長條圖、近期交易 = ✅
4. 如果看到「密碼錯誤」 → 確認 Code.gs 和輸入框的 BOSS_KEY 一模一樣（大小寫、連字號都要對）
5. 如果看到「尚未設定 BOSS_API_URL」 → URL 沒貼或沒存檔

## 🛠️ 疑難排解

### 密碼對了但卡在「讀取中...」
→ 打開瀏覽器 DevTools → Network 看 `/exec?key=...` 那一筆：
- HTTP 302 或 401：部署時「存取權」沒設成「任何人」，重新部署
- HTTP 200 但 JSON 有 `"error"` 欄：照錯誤訊息處理

### 錯誤：`找不到工作表：服務紀錄`
→ 檔案 A 沒有「服務紀錄」工作表，或名字不同（回步驟 1）

### 錯誤：`You do not have permission to call SpreadsheetApp.openByUrl`
→ `FILE_A_URL` 貼錯、或檔案 A 不在老闆帳號底下。確認是同一個 Google 帳號擁有兩份檔案。

### 金額都是 0 或怪怪的
→ 檔案 A「服務紀錄」E 欄是文字不是數字。選整欄 → 格式 → 數字 → 數字。

### 日期篩選怪怪的（沒撈到資料）
→ A 欄是文字不是日期。選整欄 → 格式 → 數字 → 日期。舊資料要重新輸入一次。

### 改了 Code.gs 之後網頁沒變化
→ Apps Script 改完**必須重新部署新版本**：部署 → 管理部署 → 齒輪 → 編輯 → 版本選「新版本」 → 部署。**URL 不會變**，不用改 index.html。

### 想換 BOSS_KEY 怎麼辦
→ Code.gs 改 `BOSS_KEY` → 存檔 → 重新部署新版本 → 打開 index.html 用新密碼登入。

---

## 🔒 安全守則（給老闆自己）

- BOSS_KEY 只存在兩個地方：**密碼管理器** + **Apps Script 裡的 Code.gs**
- 不要把 Code.gs 原始碼傳給員工、不要 commit 到 repo（這份 repo 是公開的）
- 員工的帳號：只給檔案 A 的編輯權，**不要給檔案 B 的任何權限**
- 儀表板網址（index.html）可以放 GitHub Pages 沒關係，**但 BOSS_KEY 不能寫進原始碼**
- 員工離職：改一次 BOSS_KEY + 重新部署 Apps Script（舊網址的密碼會失效）

## 📱 加到手機主畫面（像 App 一樣開）

> **前置**：儀表板必須放在 **HTTPS 網址**下（GitHub Pages 或 Firebase Hosting 都可以；file:// 或 http 不行）。
> GitHub Pages 啟用方式：repo → Settings → Pages → Source 選 `master` → Save。幾分鐘後 `https://<你的帳號>.github.io/mylittlethings/tools/pet-grooming-boss/` 就能用。

### 🤖 安卓（Chrome）

1. 用 **Chrome**（不是 Samsung Browser）打開儀表板網址並登入一次
2. 右上角「⋮」選單 → **「安裝應用程式」** 或 **「加到主畫面」**
   - 看到「**安裝**」最好 → 會變真 PWA App（獨立視窗、進 App 抽屜）
   - 只有「加到主畫面」也 OK（行為類似捷徑）
3. 確認名稱「老闆」→ 按「**安裝**」
4. 主畫面 / App 抽屜出現深藍+金色柱狀圖 icon = ✅

### 🍎 iPhone（Safari）

1. 用 **Safari**（不是 Chrome）打開儀表板網址並登入一次
2. 按底部中間的「**分享**」🔼
3. 往下滑 → 點「**加入主畫面**」
4. 名稱預設「老闆」（可改）→ 按「**新增**」
5. 回到主畫面，看到 icon = ✅

之後點 icon：
- 兩家都全螢幕開（沒網址列、沒瀏覽器 UI）
- 密碼已存在 localStorage，通常不用再輸入

### iPhone 圖示變空白怎麼辦

舊版 iOS（< 16）不認 SVG 的 `apple-touch-icon`。解法（安卓不用做）：

1. 打開 [cloudconvert.com/svg-to-png](https://cloudconvert.com/svg-to-png)
2. 丟 `icon.svg` 進去，輸出尺寸選 `180x180`
3. 下載 `icon.png`，放到 `tools/pet-grooming-boss/` 底下
4. 把 `index.html` 的這行改掉：
   ```html
   <link rel="apple-touch-icon" href="icon.svg">
   ```
   改成：
   ```html
   <link rel="apple-touch-icon" href="icon.png">
   ```
5. iPhone 上：移除舊的主畫面圖示 → 重新「加入主畫面」
