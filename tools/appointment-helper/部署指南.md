# 預約小幫手 部署指南

> 目的：每天早上打開一次，看到明天預約清單 + 客戶提醒草稿，複製貼上兩次就搞定當天通知。
> 前置：檔案 A「客戶資料」工作表已經有東西（要反查主人姓名）。
> 預計時間：15 分鐘（含測試）。

---

## 🎯 總流程

```
老闆 Google 日曆照舊記預約
  → 每個事件描述欄填客戶電話（09XXXXXXXX）
    → 建檔案 C（空試算表）
      → 檔案 C 內打開 Apps Script
        → 貼上 Code.gs
          → 改兩個常數 + 第一次授權（日曆 + Sheets）
            → 部署為網頁應用程式（任何人）
              → 拿到 /exec 網址
                → 貼到 index.html 的 APPT_API_URL
                  → 完成 ✅
```

---

## 🗓️ 前提：日曆事件怎麼寫

| 欄位 | 怎麼填 | 範例 |
|------|--------|------|
| **標題** | 寵物名；剪毛加 ✂️ 或「剪」字 | `寶寶 ✂️` / `QQ` / `大米happy` |
| **描述欄** | 客戶手機號（格式不限） | `0912-345-678` / `0912345678` / `0912 345 678` |
| **時間** | 照舊選預約時段 | 下午 12:00 ~ 1:00 |
| **排除關鍵字** | 標題含「特休 / 公休 / 休假 / 店休」自動忽略 | `萱特休` / `公休一天` |

> ⚠️ **描述欄填電話**這件事一定要做，不然系統抓不到主人姓名，該客戶就只會出現在「需要手動通知」區塊。

---

## 步驟 1：建檔案 C（預約小幫手的容器）

1. **用老闆帳號**到 [sheets.new](https://sheets.new) 建一份空白試算表
2. 命名為【**檔案 C：預約小幫手**】
3. 裡面不用填任何東西
4. **不要分享給員工**（和檔案 B 一樣理由：Apps Script 裡有密碼）

## 步驟 2：檔案 C 內打開 Apps Script，貼上 Code.gs

1. 檔案 C 裡：**擴充功能 → Apps Script**
2. 打開 `tools/appointment-helper/Code.gs`（這個 repo 內），全選複製
3. 刪掉預設的 `function myFunction() {}`，貼上整份
4. 左上專案名改成 `預約小幫手 API`
5. `Ctrl+S` 先存檔

## 步驟 3：改兩個常數（⚠️ 必改）

Code.gs 第 13-18 行：

```js
const FILE_A_URL = "https://docs.google.com/spreadsheets/d/YOUR_FILE_A_ID/edit";
const APPT_KEY = "change-me-to-random-long-string";
const CALENDAR_NAME = "";
```

### 3-1 `FILE_A_URL`
貼檔案 A 的完整網址（從 `https://` 到 `/edit`）。
這個是為了反查「電話 → 主人姓名」。

### 3-2 `APPT_KEY`
自訂長亂碼（12+ 字元），和老闆儀表板 BOSS_KEY **建議不同**。
存到密碼管理器。範例：`Appt-2026-Z7mK3q`

### 3-3 `CALENDAR_NAME`（可不改）
- **留空 `""`**：讀老闆 Google 帳號的**預設日曆**（一般人記預約都在這）
- 有在用「寵美預約」之類獨立日曆 → 填那個日曆名稱

改完 `Ctrl+S`。

## 步驟 4：第一次測試 + 授權（會要求日曆權限）

1. 函式下拉選 `testDoGet` → 按「執行」
2. **會彈出授權提示**（比儀表板多一項）：
   - 按「檢閱權限」→ 選老闆帳號
   - 「Google 尚未驗證此應用程式」→ **進階** → 前往 [你的專案名]（不安全）
   - **重點**：清單裡會看到兩項權限
     - 📝 查看和管理您的試算表（要讀檔案 A）
     - 📅 **查看您的日曆**（要讀預約事件）
   - 按「允許」
3. 執行記錄應該看到類似：
   ```json
   {"date":"2026-04-24","dateLabel":"4/24","weekday":"週五","dayWord":"明天","totalCount":0,"staffSummary":"📅 明天 4/24 (週五) 無預約",...}
   ```
4. 看到 `"error":"找不到工作表：客戶資料"` → 檢查 `FILE_A_URL`、檔案 A 工作表名
5. **先不部署**，下一步先在日曆建一筆測試事件

## 步驟 5：建一筆測試日曆事件

在 Google 日曆上，**明天**那天隨便找個時段建事件：
- 標題：`測試寶寶 ✂️`
- 時間：下午 2:00 ~ 3:00
- 描述欄：填一個**真的在檔案 A 客戶資料**裡的電話號碼（例如 `0912345678`）

回 Apps Script 再跑一次 `testDoGet` → 執行記錄應該看到：
```json
{..."totalCount":1,"reminders":[{"ownerName":"王大明","petName":"測試寶寶","time":"下午 2:00",...}],...}
```

如果 `ownerName` 是空字串 → 表示那個電話不在客戶資料裡（正常情況下會出現在 `missingPhone`）。

## 步驟 6：部署為網頁應用程式

1. 右上「**部署**」→「**新增部署**」
2. 齒輪 → 選「**網頁應用程式**」
3. 設定：
   - 說明：`v1`
   - 執行身分：**我（老闆帳號）**
   - 存取權：**任何人**（靠 APPT_KEY 密碼保護）
4. 按「部署」
5. 複製「**網頁應用程式 URL**」：
   ```
   https://script.google.com/macros/s/AKfycbx.....APPT...../exec
   ```
6. 按「完成」

## 步驟 7：把 URL 貼進 index.html

打開 `tools/appointment-helper/index.html`，找到這行：

```js
const APPT_API_URL = "";
```

改成：

```js
const APPT_API_URL = "https://script.google.com/macros/s/AKfycbx.....APPT...../exec";
```

⚠️ **不要把 APPT_KEY 寫在這裡**。密碼只有打開網頁時老闆手動輸入。

存檔。

## 步驟 8：驗證

1. 打開 index.html（雙擊或丟上 GitHub Pages）
2. 輸入 APPT_KEY
3. 看到：
   - 頂部「明天 4/24 (週五) · 1 件預約」
   - 員工清單區塊：`📅 明天 4/24 (週五) 預約 1 件\n\n下午 2:00　測試寶寶 ✂️（剪毛）`
   - 客戶提醒草稿：一張卡片，含複製按鈕
   - 按 📋 複製 → 訊息已到剪貼簿 ✅

刪掉日曆那筆測試事件。

---

## 📅 每天怎麼用

1. 前一晚或早上，打開 index.html（建議加到 Chrome 書籤 / 手機主畫面）
2. 輸入密碼（第一次後 localStorage 記住）
3. **員工清單** → 按「📋 複製全部」→ 貼到員工 LINE 群組
4. **客戶提醒草稿** → 每張卡片按「📋 複製此則」→ 貼到對應客戶 LINE
5. **⚠️ 需要手動通知** 區塊（紅橘色）→ 自己判斷要不要通知、怎麼通知

## 🛠️ 疑難排解

### 錯誤：`找不到工作表：客戶資料`
→ 檔案 A 的客戶資料表沒建或改名。回員工查詢頁的 `Sheets建置指南.md`。

### 錯誤：`You do not have permission to call CalendarApp.getDefaultCalendar`
→ 步驟 4 授權沒過「日曆」那項。重新跑 `testDoGet` 走一次授權流程。

### 員工清單是空的、但日曆明天明明有事件
→ 三種可能：
1. Apps Script 讀的是**不同日曆**（`CALENDAR_NAME` 設錯）
2. 事件標題含「特休/公休/休假/店休」被排除了
3. Apps Script 是老闆帳號，但日曆在別的帳號底下（用老闆帳號登入日曆確認）

### 客戶都跑到「需要手動通知」區塊、沒產提醒稿
→ 日曆事件**描述欄沒填電話**，或填的電話不在客戶資料表裡。

### 改了 Code.gs 後網頁沒更新
→ Apps Script 要**重新部署新版本**：部署 → 管理部署 → ✏️ → 版本「新版本」→ 部署。URL 不變。

### 想看「今天」或「後天」的預約
→ 直接打 API `?key=XXX&date=2026-04-24`（YYYY-MM-DD）測試。網頁 V1 只看明天；要新增切換按鈕在 V2 再做。

## 📈 後續可以加的功能（V2+）

- **自動寄 email**：每早 8:00 自動把「明日預約」摘要寄到老闆信箱
- **每日 trigger**：Apps Script 時間觸發條件，不用手動打開
- **LINE Messaging API**：真的自動推給客戶（需要 LINE 官方帳號 + 客戶加好友 + 綁定電話）
- **看日 / 週選擇**：今天 / 明天 / 本週一覽
- **未完成預約追蹤**：客戶 no-show 自動標記
- **PWA 主畫面 icon**：和老闆儀表板一樣加到主畫面
