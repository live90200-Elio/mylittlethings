# Cowork KEY IN 規則 — 寵美服務簽收單

給 Cowork（負責把老闆娘拍的手寫單影像識別、KEY IN 進 Google Sheets 的人或 AI 工具）的作業規則。

## 📌 一、收到照片先判斷是哪張單

**標題是辨識依據**：

| 標題（第 1 列大字） | 表單類型 | KEY IN 目標 Sheets 分頁 |
|---|---|---|
| `🐾 洗毛這件小事 — 散客服務簽收單` | 散客單 | 檔案 A「服務紀錄」分頁 |
| `🐾 洗毛這件小事 — 儲值客戶服務簽收單` | 儲值客戶單 | 檔案 B「交易明細」分頁 |

兩張單的目標分頁**完全不同**，不要混在一起 KEY IN。

## 📌 二、頁首（每張單只擷取一次）

- **日期**：頁首格式 `日期：YYYY 年 MM 月 DD 日` → 轉 `YYYY/MM/DD`（純數字，給 Sheets 的 Date 物件用）
- **美容師**：頁首格式 `美容師：XXX` → 寫到對應分頁的「美容師」欄（散客單有；儲值單沒這欄不用寫）

> 頁首日期/美容師會套用到該張單**全部 12 列**，不用每列重複問。

## 📌 三、散客單 → 檔案 A「服務紀錄」分頁

### 欄位 mapping

手寫單欄位 → Sheets 欄位（**順序很重要**）：

| 手寫單欄 | Sheets 欄 | 處理規則 |
|---|---|---|
| (頁首) 日期 | A 日期 | YYYY/MM/DD 純數字日期物件 |
| 主人電話 | B 主人電話 | 強制 10 碼字串、開頭必須是 0、保留前導 0（見「電話處理特別注意」） |
| 寵物名 | C 寵物名 | 原樣 |
| 服務項目（勾選） | D 服務項目 | 看哪幾個 ☐ 被勾 ✓/塗滿 → 用「、」串聯（例：「洗澡、剪指甲」）；勾「其他」就讀手寫補充 |
| 金額 | E 金額 | 純數字 integer（例 800）。若手寫帶 NT$/元/逗號，去除後存純數字 |
| 付款方式（勾選） | F 付款方式 | 看哪個 ☐ 被勾 → 寫該選項文字（「現金」/「匯款」/「LINE Pay」） |
| (頁首) 美容師 | G 美容師 | 套用頁首值 |
| 備註 | H 備註 | 原樣，空白就空白 |
| 簽收 | （不寫進 Sheets） | 簽名格只是客戶簽收證明，紙本歸檔即可 |

### 「#」欄不寫進 Sheets

`#` 只是手寫單的行序提示（1-12），**不要**當 Sheets 欄位寫。

### 空列要跳過

如果某列只有預印的 # 和 ☐ checkbox（沒勾、沒填電話），代表那列沒用，**整列跳過不寫**。

## 📌 四、儲值客戶單 → 檔案 B「交易明細」分頁

### 欄位 mapping

| 手寫單欄 | Sheets 欄 | 處理規則 |
|---|---|---|
| 主人電話 | A 電話 | 強制 10 碼字串、開頭 0 必留 |
| 客戶姓名 | B 客戶姓名 | 原樣 |
| (頁首) 日期 | C 日期 | YYYY/MM/DD |
| 儲值金額 | D 儲值金額 | **大多空白**。有填數字才寫；空白就空白（不要寫 0） |
| 服務項目（勾選） | E 消費項目 | 同散客單規則，串「、」 |
| 消費金額 | F 消費金額 | 純數字。若空白寫 0 |
| 扣後餘額 | G 餘額 | 純數字。若空白寫 0 |
| 簽收 | H 簽名 | 純文字寫「已簽」即可（如果有需要可填「客戶簽名: <辨識的姓名>」，但簽名通常很潦草，預設只寫「已簽」） |
| 備註 | I 備註 | 原樣 |

### 「#」欄不寫進 Sheets，空列跳過

同散客單規則。

### 三種儲值客戶情境

員工填單會有 3 種情境，KEY IN 時要看 D / F 欄哪欄有值：

| 情境 | D 儲值金額 | F 消費金額 | 對應動作 |
|---|---|---|---|
| 純消費扣抵（最常見） | 空 | 有數字 | 餘額減 |
| 純新增儲值 | 有數字 | 空 / 0 | 餘額加 |
| 邊儲值邊消費（少） | 有數字 | 有數字 | 餘額 = 原餘額 + D - F |

員工已經把「扣後餘額」算好寫在 G 欄，KEY IN 時直接照抄 G 欄即可，不用重算。但**如果** G 欄空白或數字明顯不合理（例如算錯），先 KEY IN 其他欄位 + 在備註標 `⚠️ G 欄餘額疑似有誤待老闆校對`，請老闆下班後對。

## 📌 五、電話處理特別注意 ⚠️

這是 4/28 才剛踩過的坑，請特別小心：

1. **電話必須是 10 碼字串、開頭 0 不能丟。** 例：`0905435751`、`0988969115`。
2. 手寫可能有破折號或空格（`0905-435-751` / `0905 435 751`），KEY IN 時清乾淨成純 10 碼字串：`0905435751`。
3. **KEY IN 進 Sheets 時，務必把 A 欄該 cell 設成「純文字」格式再寫值**，否則 Sheets「自動」格式會把純數字字串 cast 成 number、吃掉開頭 0（變 905435751，9 碼）。
   - Apps Script 寫法：`sheet.getRange(row, col).setNumberFormat("@").setValue(phone)`
   - 手動 KEY IN：先選整欄 A → 格式 → 數字 → 純文字
4. **若手寫電話只有 9 碼、缺開頭 0**（員工漏寫或老闆娘看錯），KEY IN 時自動補上 0 → 但**同時在備註欄標記** `⚠️ 電話補 0`，方便後續校對。

## 📌 六、金額處理

- **純數字 integer**，不要小數、不要逗號、不要 NT$、不要「元」
- 例：手寫 `1,500元` → KEY IN `1500`
- 例：手寫 `NT$ 800` → KEY IN `800`
- 空白或無法辨識：留空（不要填 0），備註欄寫 `⚠️ 金額待校對`

## 📌 七、勾選符號的判讀

員工被告知用 ✓ 或塗滿，但實際可能多種寫法。判讀規則（從強到弱）：

1. **塗滿成黑色方塊** ■ → 確定勾選
2. **裡面有 ✓ 打勾** → 確定勾選
3. **裡面有點點 / 一槓 / 圈圈** → 視為勾選
4. **完全空白 ☐** → 沒勾選
5. **斜線 ☑（不確定的塗鴉）** → 視為勾選但**備註欄標** `⚠️ 勾選符號不明`

> 預印的 ☐ 是空格符號（Unicode `☐`）— 沒被加任何記號就是「沒勾」。

## 📌 八、KEY IN 完成後檢查清單

寫進 Sheets 之前最後確認：

- [ ] 電話都是 10 碼字串、開頭 0 都在
- [ ] 金額都是 integer，沒有 NT$ / 元 / 逗號
- [ ] 日期是 Date 物件不是純文字（Sheets 的營收報表 trigger 會用 `instanceof Date` 判斷，純文字日期會被跳過）
- [ ] 散客單寫進「服務紀錄」、儲值單寫進「交易明細」（不要寫錯分頁）
- [ ] 空列跳過了沒
- [ ] 有 `⚠️` 備註的列已彙整成 list 給老闆娘人工校對

## 📌 九、回報

KEY IN 完跟老闆娘回報：
```
✅ YYYY/MM/DD 服務紀錄 KEY IN 完成
- 散客單：N 筆 → 服務紀錄分頁
- 儲值客戶單：M 筆 → 交易明細分頁
- 待校對：K 筆（詳列電話 + 原因）
```
